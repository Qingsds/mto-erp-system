# Parts 模块规则

适用页面：

- `apps/frontend/src/pages/parts/list/PartsPage.tsx`
- `apps/frontend/src/pages/parts/manage/PartForm.tsx`
- `apps/frontend/src/pages/parts/detail/PartDetailPage.tsx`

## 1. 列表页

- 搜索优先覆盖零件名称、编号，后续可扩展材质与规格
- 价格展示优先展示“标准价”，没有时再回退其他价格
- “新增零件”和“批量导入”保持为主操作组，不拆散到多处
- 空状态必须提供“新增零件”或“批量导入”入口

## 2. 详情页信息结构

固定为：

1. 左侧图纸区
2. 右侧零件基础信息
3. 价格区
4. 历史图纸版本

规则：

- 图纸区是详情页第一视觉重点，不应退化为附件链接列表
- 右侧信息区优先就地编辑，不优先弹窗编辑
- 历史版本要可直接查看和下载，不能只显示文件名

## 3. 图纸区

- 只混合“图片图纸”和“PDF 图纸”，不把加工代码并入同一交互
- 图片优先页内预览
- PDF 提供页内查看或清晰的查看/下载入口
- 文件限制、允许格式、版本策略要在上传前明确可见
- 上传失败必须在图纸区内展示，不只 toast

## 4. 移动端

- 图纸预览与上传入口应在首屏上半区
- 历史版本折叠展示，默认只展开最新版本
- 若存在底部操作区，按钮数量控制在 2 个以内
