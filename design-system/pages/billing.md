# Billing 模块规则

适用页面：

- `apps/frontend/src/pages/billing/BillingPage.tsx`

## 1. 页面定位

Billing 不是纯展示页，而是“对账单创建 + 盖章 + 状态流转”的操作页。

因此页面必须同时承载：

- 状态筛选
- 卡片摘要
- 写操作入口
- 状态变更动作

## 2. 卡片信息结构

每张卡片至少应稳定展示：

- 对账单号
- 客户名
- 状态
- 金额
- 条目数量或关联发货单数量
- 创建时间

规则：

- 状态标签始终靠近卡片头部
- 金额是主值，不要弱化
- 操作区与信息区分层，避免按钮混进正文

## 3. 写操作入口

- “新建对账单”是页面级主操作，固定放标题区
- “盖章”“标记已结清”等状态动作放卡片底部操作区
- 状态变更前需明确当前状态和目标状态

## 4. 空状态与错误状态

- 空状态必须提供“新建对账单”入口
- 若写操作失败，应在当前上下文看到具体失败原因
- 卡片操作 loading 只锁定当前卡片，不冻结全页

## 5. 后续增强边界

- 月度汇总、待收金额等统计可补在标题区下方
- 不建议把 Billing 页面做成高装饰 Dashboard
