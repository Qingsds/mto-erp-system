# Orders 模块规则

适用页面：

- `apps/frontend/src/pages/orders/OrdersPage.tsx`
- `apps/frontend/src/pages/orders/OrderDetailPage.tsx`

## 1. 列表页

- 状态分段是主筛选，不要弱化
- 状态项优先显示数量，帮助判断工作量
- 搜索应优先覆盖订单号与客户名；如果当前只支持客户名，文案必须明确
- 筛选变化应尽量保留在 URL 或页面状态中，避免刷新即丢失

## 2. 订单详情页

顶部身份区固定包括：

- 订单号
- 主状态
- 主操作

摘要区优先展示：

- 发货进度
- 金额
- 待发数量
- 订单创建时间或交付关键信息

## 3. 明细与记录

- 订单项表格优先强调“已发 / 待发 / 总量”
- 发货记录优先做时间线或结构化列表，不建议无分组平铺
- 详情 tab 仅用于平级内容切换，不要把关键主操作藏入 tab

## 4. 高风险操作

- 短交结案必须二次确认
- 原因输入不能写死为默认文案后直接提交
- 危险动作需要说明对金额、数量、后续流程的影响

## 5. 移动端

- 底部操作栏必须贴近底部导航上方
- 超过两个动作时，次要操作收进更多菜单
- 详情页首屏先给进度与状态，再给明细
