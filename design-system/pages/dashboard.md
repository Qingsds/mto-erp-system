# Dashboard 模块规则

适用页面：

- `apps/frontend/src/pages/dashboard/DashboardPage.tsx`

本页目标不是品牌展示，而是“当天业务概览 + 快速判断优先事项”。

## 1. 信息优先级

从高到低固定为：

1. 核心统计
2. 待处理事项
3. 最近订单/发货动态
4. 趋势或结构型图表

规则：

- 统计卡必须服务于判断，不做纯装饰数字
- `delta`、趋势箭头、时间基准要能解释变化
- 待办项应支持跳转，不只做文案列表

## 2. 统计卡

- 桌面端最多一行 4 张，移动端 2 列
- 卡片高度尽量统一，避免信息跳动
- 主值、状态趋势、说明文案三层足够，不叠加额外小组件
- 移动端不要把统计卡文字压到 `text-[10px]`

## 3. 主内容区

- 当前如果同时存在表格和趋势信息，优先保留业务表格，再为趋势补一个次级区域
- 图表只在能帮助发现异常时引入，不为了“更像 dashboard”硬塞复杂图表
- 订单进度适合使用进度条，但要同时显示数值结果

## 4. 移动端

- 顶部问候语与日期说明要收敛，避免抢占有效区域
- 最近订单卡片以状态、客户、金额、进度为主，不重复桌面端全部字段
- 移动端首页的主操作入口应可直接跳到订单、发货、对账等高频页面
